
# Street Food Decoder 🌮

A prototype mobile/web application designed for university evaluation. The project focuses on solving real-world language and dietary barriers for travelers by providing a digital solution to scan and decode local street food menus.

---

## 🎓 Academic Project Overview

This repository contains the source code for the **Street Food Decoder** system, developed as a core component of the curriculum evaluation. The primary objective is to demonstrate practical software engineering principles, UI/UX design, and database integration within a structured development lifecycle.

---

## 🛠️ Project Core Architecture

The system is built using a straightforward, decoupled architecture to ensure clean separation of concerns:

- **Frontend (Presentation Layer):** Handles the user interface, image submission, and displaying the translated/decoded food information to the user.
- **Backend (Logic Layer):** Processes user input, handles communication requests, and fetches data from the storage layer.
- **Data Store (Storage Layer):** Holds the predefined dataset of food items, ingredients, translations, and allergen alerts.

---

## 🚀 Key Functional Features Implemented

1. **Food Item Lookup/Scan Simulation:** Allows users to input or capture a dish to retrieve instant cultural context and descriptions.
2. **Allergen & Dietary Filter:** Filters out ingredients based on user preferences (e.g., Vegetarian, Nut Allergies).
3. **Weekly Metrics Tracker:** Business logic optimized to manage and calculate estimated weekly tourist volumes and data interactions.
